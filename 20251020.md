# 16~18 원격 수업 정리

## 16 원격 저장소 복제 Clone

### 원격 저장소 생성과 복제
- $ git clone [복사된-주소] : 원격저장소와 동일한 이름으로 복제
- $ git clone [복사된-주소] [새로운-폴더명] : 하부 폴더 [새로운-폴더명] 이름으로 복제
- $ git clone [복사된-주소]. : 현재 폴더에 바로 복제

### 원격 저장소 별칭 이름 점검
- $ git remote : 원격 저장소 목록
    - 기본 이름 origin 점검
    - 위에서 clone을 했기 때문에 별칭 origin이 위의 주소가 설정
- $ git remote -v : 저장소 주소 등 원격 저장소 정보 목록 표시

### 원격 저장소 별칭 관리
- $ git remote add origin URL : 원격 저장소 별칭 저
- $ git remote show origin : 자세한 정보
- $ git remote rename origin org : 이름 수정
- $ git remote rm org : 삭제

### $ git remote show origin(atom)
- 원격 저장소의 모든 브랜치 보기 가능

### vscode로 원격 저장소 관리
- Clone git repository 클릭

## 17 지역과 원격 저장소 연동 push pull

### 원격 저장소 복제 연동
- $ git clone http://github.com/atom/atom.git

### 원격 저장소 수정 사항 pull로 지역 저장소로 가져오기
- $ git pull origin main
- $ git pull

### 지역 저장소에서 push
- 쓰기 권한이 있어야 push 가능
- $ git push <저장소별칭명> <브랜치명>
- $ git push origin topic : 최초에 한 번만 브랜치 명을 입력하고 그 이후에는 모든 인자를 생략 가능
    - 성공 이후 간단히 $ git push 명령어로 가능

### git pull 명령
- fetch 명령과 병합하는 merge 명령이 순차적으로 진행
- fetch 명령 : 원격 저장소의 정보를 로컬(remote/origin) 저장소로 가져오는 명령
- merge 명령 : 변경된 정보를 로컬 저장소의 내용과 병합

### fetch
- 원격 저장소에서 로컬 저장소로 소스만을 가져와 병합을 미수행하는 명령
- $ git fetch <remote>
    - $ git fetch origin
- fetch 이후 origin/main, origin/HEAD가 더 최신의 위치에 있음
    - 지역인 HEAD의 main이 이후에 있음
- 깃허브의 내용을 반영하려면 fetch 후 merge 해야함
    - $ git merge origin/main
