# 22~24 원격 수업 정리

## 22 다양한 브랜치 병합

### 병합(merge)
- 두 개의 브랜치를 하나로 모으는 과정
  - fast-forward(빨리 감기) 병합
  - 3-awy 병합

### fast-forward 병합
- 조건
  - 현재 브랜치 master가 병합할 대상 커밋의 직접적인 뿌리가 되는 경우
    - 두 브랜치가 일렬 상태

### 3-way 병합
- 새로운 커밋을 사용하여 두 기록을 합치는 것

### 병합의 다양한 옵션
- $ git merge {병합할 브랜치 명}
  - 보통의 병합
  - 현 브랜치와 병합할 브랜치가 일렬 상태 :  fast-forward 병합
  - 현 브랜치와 병합할 브랜치가 갈라져 있는 상태 : 3-way
- $ git merge --no-ff {병합할_브랜치_명}
  - 무조건 3-way 병합되는 옵션
- $ git merge --ff-only {병합할_브랜치_명}
  - 상태가 fast-forward인 일렬 상태에서만 병합 진행
- $ git merge --squash {병합할_브랜치_명}
  - 현재 브랜치에 병합 대상과의 합치는 커밋을 하나 생성해 병합
    - 병합되는 브랜치는 사용하지 않고 그대로 남음

## 24 병합 충돌과 해결

### 병합 충돌 기준
- 파일이 수정되지 않거나 한쪽 브랜치에서만 수정되면 파일 충돌 없음
- 두 브랜치 모두에서 변경 사항이 달리 발생한 파일의 경우 파일 충돌 발생

#### 3-way 충돌 발생
- $ git merge hotfix

### 충돌 해결
- 충돌이 발생하면 해당 파일의 충돌을 먼저 해결해야 함.
  - 직접 파일 내용을 수정 후 저장
    - 충돌 표시 모두 제거
  - 계속해서 add, commit 진행 : $ git commit -am 'msg
  - 필요할 경우 합병된 이전 브랜치 삭

### 병합 취소 후 다시 병합
- $ git merge --abort : 병합 취소
- $ git merge feat/list : 다시 병합
