# 10~12 원격 수업 정리

## 10 파일 비교 diff

### 스테이징 영역 기준으로 작업 디렉토리 비교
- $ git diff

### 깃 저장소 HEAD 기준으로 스테이징 영역 비교
- $ git diff --staged [HEAD]

### 깃 저장소 HEAD 기준으로 작업 디렉토리 비교
- $ git diff HEAD
- $ git diff HEAD~2 : HEAD~2 기준

### 커밋 간의 파일 비교
- git diff [비교할 commit해쉬1] [비교할 commit해쉬2]
    - git diff HEAD^ HEAD
 
## 11 파일 삭제 rm과 복원 restore

### 리눅스 명령 파일 삭제
- $ rm [file] : 작업 디렉토리에서 file 삭제

### 깃 명령 파일 삭제
- $ git rm [file] : 작업 디렉토리+스테이징 영역에서 모두 file 삭제
    - 다음 커밋에서 지정한 file을 삭제하겠다는 의미
    - Tracked 상태의 파일을 제거하여 Untracked 상태로 만듦
- $ git rm --cached [file]
    - 스테이징 영역에서 file 삭제 -> 작업 디렉토리에서는 삭제안됨
    - $ git ls files 결과에서 보이지 않음 -> 기본적으로 스테이징 영역의 파일 목록을 표시
 
### 파일복원 restroe
- $ git restore [file] : 작업 디렉토리의 파일 [file]을 스테이징 영역의 파일 상태로 복구
    - 작업 디렉토리에 있던 [file] 내용이 사라지므로 유의, 파일 [file]이 작업 디렉토리와 스테이징 영역이 같은 상태
- $ git restore --staged : 깃 저장소의 최신 커밋 상태의 파일 [file]를 스테이징 영역에 복구
    - 스테이징 영역에 있던 [file] 내용이 사라지므로 유의, 스테이징 영역의 파일 [file]이 깃 저장소 영역과 같은 상태가 
- $ git restore --source=HAED --staged --worktree [file]
    - 파일 [file]가 현재 커밋 상태의 내용으로 작업 디렉토리와 스테이징 영역 모두 같은 상태가 됨
- $ git restore --source==HEAD^ staged [file]
    - 스테이징 영역의 파일 [file]가 깃 저장소 영역과 같은 상태가 됨
