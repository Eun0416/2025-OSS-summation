# 25~27 원격 수업 정리

## 25 브랜치 리베이스 rebase

### 3-way 상태에서 base의 이해
- 'master' 브랜치 커밋 B에서 분기되는 'bugfix' 브랜치
  - 커밋 B
  - 현재 master와 bugfix의 공통조상 : base

### rebase 이해
- base를 수정
  - B에서 마스터의 최신 커밋인 D로 수정
- 이후 다시 'fast-forward 병합' 직접 다시 수행
  - 'master' 브랜치의 HEAD가 'bugfix' 브랜치 마지막 커밋으로 이동
    - 로그 이력이 분기 없이 하나의 줄기로 이어짐
